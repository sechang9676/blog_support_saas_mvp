# GOGIF AdSense 승인용 콘텐츠/페이지 구축 플랜

## 1. GOGIF 애드센스 승인 전략 요약

GOGIF는 단순 변환 툴이 아니라, 사용자에게 실제로 도움이 되는 설명 콘텐츠와 명확한 사이트 구조를 함께 제공하는 웹사이트로 보여야 합니다. AdSense 심사에서는 기능만 있는 1페이지 툴보다, 어떤 서비스인지 설명하는 콘텐츠, 정책 페이지, 문의 페이지, 내부 링크 구조가 분명한 사이트가 더 안정적으로 보입니다.

핵심 전략은 다음과 같습니다.

- 메인 페이지에 변환 도구와 설명형 랜딩 콘텐츠를 함께 배치합니다.
- 소개, 사용법, 블로그, 개인정보처리방침, 이용약관, 문의 페이지를 모두 노출합니다.
- 블로그에는 독창적인 한국어 설명 글을 넣어 “가치 있는 콘텐츠” 신호를 강화합니다.
- 모든 페이지에서 Header/Footer를 공통으로 유지해 사이트 구조를 명확히 합니다.
- 푸터에는 `About`, `How to Use`, `Blog`, `Privacy Policy`, `Terms`, `Contact` 링크를 항상 보이게 합니다.
- 개인정보처리방침에는 Google 및 제3자 광고 쿠키, 맞춤형 광고, 광고 개인 최적화 해제 안내를 포함합니다.

## 2. 현재 문제점

- 1페이지짜리 툴 사이트
  - 메인 기능만 있으면 사용자에게는 유용해도, 심사 관점에서는 “도구만 있고 설명이 부족한 사이트”로 보일 수 있습니다.
- 콘텐츠 부족
  - 도구 외에 읽을 수 있는 문서, 가이드, 활용 팁이 없으면 no content 또는 valuable inventory 이슈가 생길 수 있습니다.
- 신뢰 페이지 부족
  - 개인정보처리방침, 이용약관, 문의 페이지가 없으면 운영 주체와 정책이 불분명해 보일 수 있습니다.
- 검색엔진용 구조 부족
  - 내부 링크, 페이지 계층, 제목 구조가 약하면 검색엔진과 심사 시스템이 사이트 주제를 이해하기 어렵습니다.

## 4. 각 페이지의 목적

### `/`

- GOGIF의 첫 인상과 핵심 기능을 전달합니다.
- 변환 도구를 제공하면서도, 사이트가 “설명 콘텐츠가 있는 서비스”처럼 보이도록 만듭니다.
- 관련 글, FAQ, 제작 팁을 함께 배치해 체류 시간을 늘립니다.

### `/about`

- GOGIF가 어떤 서비스인지 설명합니다.
- 누구에게 도움이 되는지, 왜 이 도구가 필요한지 신뢰감 있게 전달합니다.

### `/how-to-use`

- 업로드, 설정, 다운로드의 순서를 안내합니다.
- 초보자가 바로 따라 할 수 있도록 단계를 명확히 보여줍니다.

### `/blog`

- 3개의 핵심 글을 카드 형태로 보여줍니다.
- 사이트가 단순 기능 페이지가 아니라 콘텐츠 허브처럼 보이게 합니다.

### `/blog/video-to-gif-guide`

- 동영상을 GIF로 만드는 가장 쉬운 방법을 설명합니다.
- GOGIF 사용 흐름을 자연스럽게 포함합니다.

### `/blog/reduce-gif-size`

- GIF 용량을 줄이는 실전 기준을 설명합니다.
- 파일 최적화에 대한 독창적인 정보 페이지 역할을 합니다.

### `/blog/gif-for-social-media`

- SNS, 블로그, 상세페이지, 메신저에서 GIF를 어떻게 쓸지 안내합니다.
- GOGIF의 사용 맥락을 확장해 보여줍니다.

### `/privacy`

- 개인정보, 쿠키, Google AdSense, 제3자 광고 쿠키, 맞춤형 광고 관련 안내를 제공합니다.
- 심사에서 가장 중요한 신뢰 페이지 중 하나입니다.

### `/terms`

- 서비스 목적과 이용 규칙을 명확히 합니다.
- 저작권 침해 콘텐츠 업로드 금지와 면책 조항을 포함합니다.

### `/contact`

- 사용자 문의 경로를 제공합니다.
- 오류 제보, 기능 제안, 광고/제휴 문의, 개인정보 문의를 분리해서 안내합니다.

## 9. 애드센스 신청 전 체크리스트

- 개인정보처리방침과 이용약관이 푸터에서 항상 보이는지 확인
- 문의 페이지가 실제로 접근 가능한지 확인
- 모든 페이지의 title과 description이 채워져 있는지 확인
- 404 또는 빈 페이지가 없는지 확인
- 모바일에서 헤더, 푸터, 카드 레이아웃이 깨지지 않는지 확인
- 메인 페이지에 단순 변환기 외의 설명 콘텐츠가 충분한지 확인
- 블로그 글이 얇은 복붙 문구가 아니라 GOGIF에 맞는 독창적 설명인지 확인
- 광고 영역을 비워두거나 “광고 준비중” 같은 박스를 넣지 않았는지 확인
- AdSense 관련 개인정보 고지에 Google 및 제3자 광고 쿠키, 맞춤형 광고, 최적화 해제 안내가 포함됐는지 확인

## 10. 구현 우선순위

1. 공통 Header/Footer와 라우팅 구조 정리
2. `/privacy`, `/terms`, `/contact` 같은 신뢰 페이지 우선 추가
3. `/about`, `/how-to-use`, `/blog` 목록 추가
4. 블로그 상세 글 3개 작성
5. 메인 페이지에 설명형 섹션과 FAQ 추가
6. SEO 메타 타이틀/디스크립션 적용
7. 모바일 반응형과 내부 링크 점검
8. Vercel 같은 정적 배포 환경의 라우팅 fallback 확인
